# Google Play 비공개 테스트 업데이트 정리

최종 업데이트: 2026-05-05  
앱: `로또될각`  
Android package: `com.meetyuuu.lottoanalyzer`  
현재 테스트용 AAB: `dist/lottoanalyzer-android-v4.aab`

## 지금 할 수 있는 일

QR 실제 샘플이 아직 없어도 Google Play 비공개 테스트 업데이트는 진행할 수 있다. 비공개 테스트는 완성본 제출이 아니라 테스터에게 새 기능을 배포하고 피드백을 받는 단계이기 때문이다.

이번 업데이트는 다음 목적에 맞다.

- 현재까지 구현한 기능을 테스터에게 배포
- `내 번호`, QR 스캐너 진입, 명당 지도, 최신 데이터 자동화 흐름 QA
- Google Play 비공개 테스트 업데이트 이력 확보
- 다음 QR 파서 보정 업데이트 전 중간 릴리즈로 사용

## 업로드 순서

1. Google Play Console 접속
2. `로또될각` 앱 선택
3. `테스트 > 비공개 테스트` 이동
4. 현재 사용 중인 트랙 선택
   - 기존 메모 기준: `Doply Test` 트랙 유지
5. `새 버전 만들기` 또는 현재 초안 릴리즈 편집
6. `dist/lottoanalyzer-android-v4.aab` 업로드
7. 출시명 입력
8. 출시 노트 입력
9. 권한/정책 경고 확인
10. `검토` 후 비공개 테스트 트랙으로 출시

## 업로드 후 확인

- 릴리즈 상태가 `검토 중`, `게시 대기 중`, `사용 가능` 중 무엇인지 확인
- 관리형 게시가 켜져 있으면 승인 후 별도 `게시` 버튼이 필요할 수 있음
- 테스터에게 업데이트가 보이기까지 몇 시간 걸릴 수 있음
- 테스터는 opt-in 상태여야 테스트 버전을 받을 수 있음
- 같은 사용자가 여러 테스트 트랙에 있으면 가장 높은 versionCode가 배포될 수 있음

## 이번 업데이트에서 테스트할 기능

### 공통

- 앱 실행
- 하단 탭 간 이동
- 광고 배너 영역 표시 여부
- 앱 아이콘과 splash 확인

### 번호추천 / 고정추천 / 합계생성

- 최신 당첨번호 1222회 반영 확인
- 추천 버튼 동작 확인
- 고정번호 추천 입력/결과 확인
- 합계 범위 추천 입력/결과 확인

### 내 번호

- 수동 등록 저장/삭제 확인
- 저장 목록에 `수동`, 게임 수, 저장일 표시 확인
- QR 스캔 버튼이 카메라 화면으로 진입하는지 확인
- 카메라 권한 허용/거절 흐름 확인
- 실제 QR 샘플이 없으면 QR 파싱 결과까지는 보류

### 명당

- 명당 탭 진입 시 위치 권한 요청 확인
- `판매점`, `내 주변 명당`, `전국 명당` 전환 확인
- 리스트/지도 전환 확인
- 내 위치 아이콘 재요청 흐름 확인
- 전국 순위/지역별/최근 회차 판매점 리스트 확인

### 통계

- 통계 화면 진입 확인
- 번호/합계/홀짝 통계 표시 확인
- 최신 데이터 기준으로 깨지지 않는지 확인

## 테스터에게 보낼 안내 문구

```text
로또될각 비공개 테스트 새 버전을 올렸습니다.

이번 버전에서는 내 번호 저장, QR 스캔 화면, 명당 지도, 최신 당첨번호 자동 업데이트 구조가 추가되었습니다.

확인 부탁드릴 부분:
1. 앱 설치/업데이트가 정상적으로 되는지
2. 번호추천, 고정추천, 합계생성, 통계 화면이 정상 동작하는지
3. 내 번호 탭에서 수동 저장/삭제가 되는지
4. QR 스캔 버튼을 눌렀을 때 카메라 권한과 스캔 화면이 정상적으로 뜨는지
5. 명당 탭에서 위치 권한과 지도/리스트 전환이 정상인지

오류가 있으면 사용 중인 기기명과 화면 캡처를 함께 보내주세요.
```

## 다음 업데이트 후보

다음 AAB 업데이트는 QR 실제 샘플을 받은 뒤 진행하는 것이 좋다.

후보 내용:

- 실제 로또 QR URL 기반 파서 보정
- QR 스캔 후 자동 저장 검증
- 스토어 등록 이미지/스크린샷 교체
- 명당 지도 실기기 버그 수정
- 광고 노출/레이아웃 QA 반영

예상 다음 versionCode:

```text
5
```

EAS production build는 `autoIncrement`가 켜져 있으므로 다음 빌드 때 versionCode가 자동 증가할 가능성이 높다.

## 주의사항

- 새 AAB는 기존보다 높은 versionCode여야 한다.
- Play Console에서 이미 초안 릴리즈가 남아 있으면 새 릴리즈 생성이 막힐 수 있다.
- 비공개 테스트 업데이트도 Google 심사/처리 시간이 걸릴 수 있다.
- 광고가 포함되어 있으므로 Google Play의 광고/광고 ID 선언과 AdMob 설정이 맞아야 한다.
- 실제 production 신청 전에는 스토어 이미지, 개인정보, 앱 콘텐츠, 연령 등급을 다시 확인한다.
