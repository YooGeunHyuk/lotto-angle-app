# 걷기 앱 수익화 모델 — 아이디어 루프 결과

> 작성일: 2026-07-13 · 방법: 생성 → 적대적 심사(VC 3인 패널) → 개선 루프(2회) → 최종 실행 심사
> 규모: 6개 각도 × 컨셉 생성 → 12개 후보 → 각 3표 채점 → 강한 것만 살아남기 루프 (총 93개 에이전트)
> 채점 5축(각 10점, 50점 만점): 수익잠재력 · 다운로드/바이럴 · 실현가능성 · 방어가능성 · 한국적합성

## 결론 먼저 (가장 중요)

루프를 끝까지 돌린 결과, **38점(강함) 문턱을 넘은 컨셉은 하나도 없었습니다.** 이건 실패가 아니라 신호입니다. 심사역들이 모든 상위 후보에서 **똑같은 두 가지 약점**을 반복해서 지적했습니다.

1. **방어가능성이 구조적으로 약하다 (모든 후보 def 2~4점/10점).** 걷기 앱의 기능(만보기·명당지도·리워드)은 전부 복제 가능한 공개 데이터/표준 기능이라, 트래픽이 붙는 순간 캐시워크·버즈빌 같은 기존 강자가 그대로 베낄 수 있다. **신규 진입자가 "광고-리워드 모델로 캐시워크를 이긴다"는 건 지는 게임이다** — 광고 인벤토리·제휴처·영업조직이 전부 열세이기 때문.
2. **로또 앱에 묶는 순간 '사행성 규제'가 존재 리스크가 된다.** 걸음→현금 추첨, 복권 구매 연결 같은 훅은 다운로드는 잘 되지만, 앱스토어 심사 거절 또는 사후 삭제 한 방에 유일한 유통 경로를 잃는다. 이건 매출 리스크가 아니라 **존폐 리스크**다.

그래서 루프가 내린 실질적 처방은 **"광고-리워드 모델을 버리고, 이미 존재하는 고관여 오디언스에게 '구독'을 파는 단일 축"** 입니다. 걷기·리워드·명당은 수익원이 아니라 **리텐션/바이럴 장치**로만 쓴다.

**추천 순위:**
- **실행 1순위 (규제 안전 + 깨끗한 구독): 효도 건강 리포트.** 유일하게 사행성 리스크가 없고 구독만으로 손익이 서는 모델.
- **트래픽 후크 1순위 (검증 먼저): 명당 웹 공유카드.** 코드 거의 없이 "명당 미끼가 카톡 바이럴을 만드는가"를 2주에 검증.
- **고위험/고수익: 로또워크(걸음→추첨권).** 다운로드 폭발력은 최고지만 규제·스토어 심사가 GO의 전제조건.

## 최종 3개 컨셉 (심사 통과 상위)

### 1. 운수길 인사이드 — 로또앱 내장형 + 편의 구독 (28/50, 최고점)

- **한 줄:** 별도 앱을 먼저 내지 않고 기존 로또앱 안에 '운수길 모드'로 내장해 CAC≈0으로 검증하고, 로또 편의 구독을 1차 수익으로 삼는 2단계 구조.
- **킬러 기능:** (a) **무설치 웹 공유카드** — "우리 동네 로또 1등 명당 + 오늘의 운수길"을 설치 없이 열리는 카톡 공유카드로 만들어 바이럴 → 앱 전환. (b) **자기금융형 리워드 캡** — 그날 광고+제휴 순수익의 고정 비율(예 35%)만 리워드 예산으로 자동 배정. 리워드가 수익을 수학적으로 절대 초과 못 하는 구조.
- **수익화:** 주력은 **로또 편의 구독(월 3,900원)** — 회차 알림·QR 당첨 자동확인·번호 관리·걷기 코칭. 이미 지불의사 있는 로또 오디언스라 헬스앱 평균(1% 미만)보다 높은 전환 기대. 광고 리워드는 4주 코호트로 eCPM이 원가를 덮는 게 증명된 뒤에만 확장.
- **12개월 월매출 시나리오:** 보수 250만~400만 / 기본 700만~1,200만 / 낙관 1,700만~2,400만원.
- **⚠️ 심사 지적:** 원안이 넣었던 "동행복권 제휴 수수료 월 500만~1,000만"은 **허수** — 정부독점 온라인복권 구조상 제3자 앱에 주는 구매연결 커미션 프로그램이 사실상 없음. 이 라인은 손익에서 빼야 함.
- **최대 리스크:** 복권 인접 규제 + 앱스토어 도박 카테고리 심사. 대응: 구매 유도를 전부 빼고 순수 정보·편의(당첨확인·알림·걷기)만 남긴 버전으로 심사 통과 범위를 법률검토로 먼저 확정.

### 2. 로또워크(LottoWalk) — 걸음으로 뽑는 무료 추첨권 (26/50, 다운로드 잠재력 최고)

- **한 줄:** 만보를 채우면 캐시(70원)가 아니라 '추첨 응모권'을 주고, 매일 저녁 8시 로또식 추첨으로 소수에게 몰아주는 변동보상형 걷기 앱.
- **킬러 기능:** **걸음 = 무료 추첨권.** 고정 100원(지루함) 대신 응모권 1장 → 저녁 8시 '동네 추첨쇼'로 소수에게 5만~100만원. **변동보상(도박적 기대감)이 고정 100원보다 리텐션당 원가가 훨씬 싸면서 중독성은 강하다.**
- **수익화:** **광고 재고를 사지 않고 임대한다** — 버즈빌·애드팝콘 오퍼월 SDK를 얹어 국내 최대 리워드 광고 네트워크 재고를 즉시 확보. 상금 재원은 총 광고매출의 20~30%로 캡(적자 불가 구조).
- **12개월 월매출(원가 차감 전):** 보수 DAU 8천×12원 ≈ 290만 / 기본 DAU 3만×20원 ≈ 1,800만 / 낙관 DAU 8만×32원 ≈ 7,700만원.
- **⚠️ 심사 지적:** 원안의 "DAU 20만·연 29억, ARPDAU 40원"은 낙관을 다시 2~3배 초과한 값. SDK 임대 시 네트워크 마진·프리미엄 딜 제외로 실측 블렌디드 ARPDAU는 **12~25원**으로 봐야 함. 기본 시나리오는 원안의 1/5~1/8.
- **최대 리스크(존폐급):** "걸음→현금 무작위 몰아주기"는 사행행위규제·경품고시·복권 규제 저촉 소지. 대응: 현금 대신 **기프티콘·포인트 확정 지급형**으로 설계하거나 경품 한도 내로 캡, 변호사 자문, 광고 네트워크 정책팀 사전 확인 — 이 셋 통과 못 하면 킬러 기능을 구조적으로 바꿔야 함.

### 3. 효도 건강 리포트 — 부모 걷기 케어 구독 (26/50, ★규제 안전 추천)

- **한 줄:** 떨어져 사는 부모의 걷기·활동 데이터를 자녀가 매일 확인하고, 이상 징후를 알려주는 시니어 안부+건강 구독 앱.
- **킬러 기능:** 부모가 걷기만 하면 자녀 폰으로 **"오늘 아버지 4,200보 · 어제보다 급감 · 3일째 외출 없음"** 안부 카드 자동 전송. **걷기 데이터를 '건강'이 아니라 '살아있다는 안부 신호'로 재정의** — 캐시워크가 못 건드리는 감정적 페인(원격 돌봄 불안)을 정면으로 찌름.
- **수익화:** **자녀가 결제하는 구독(부모 무료).** 베이직 무료 / 프리미엄 월 6,900원(일일 AI 안부 리포트, 이상징후 알림, 병원용 월간 PDF, 부모 2명). **광고 인벤토리 불필요 → 대형 영업조직 없이 성립.** 시니어 건강기능식품·혈압계 제휴 커머스 부가.
- **왜 다운로드:** 캐시워크 이용자 40%가 50대 이상 = 그들의 **자녀(30~50대)라는 결제력 있는 2차 시장**의 존재 증명. 부모는 리워드로, 자녀는 효(孝) 감성으로 각각 설치·결제 동기. 명절·연말 카톡 바이럴에 최적.
- **12개월 월매출:** 보수 80만~150만 / 기본 300만~600만 / 낙관(보험사·통신사·지자체 유통 제휴 1건 전제) 1,500만~2,500만원.
- **⚠️ 심사 지적:** 원안의 "월 3,300만원·연 8억대"는 낙관 시나리오의 천장. 제휴 커머스 월 500만도 트래픽 없이는 비현실적(월 50만~150만이 현실).
- **최대 리스크(단일 실패점):** **시니어 측 지속 실행 장벽.** 앱 리터러시 최저인 시니어가 데이터 소스이자 필수 활성 대상이라, 부모가 폰을 두고 나가거나 앱이 꺼지면 신호가 끊긴다. 그러면 안부 카드가 오히려 "3일째 신호 없음 → 사고?"라는 거짓 불안을 유발 → 해지. 대응: 초기엔 '안부/추세'로만 포지셔닝하고 의료 판단은 배제(의료법 리스크 회피).

## 각도별 핵심 인사이트 (수익화 패턴)

- **광고-리워드:** 신규자는 광고 재고를 **소유하지 말고 임대**하라(버즈빌·애드팝콘 오퍼월 SDK). 단 실측 ARPDAU는 캐시워크(약 51원)의 절반 이하(12~25원)로 잡을 것.
- **구독-AI:** **결제자와 사용자를 분리**하면 시니어 결제 저항을 우회할 수 있다(부모는 무료, 자녀가 결제). 광고 없이 구독만으로 손익이 서는 유일한 안전 경로.
- **바이럴-소셜:** 판돈/대결/지역 대항은 다운로드 폭발력이 크지만, "캐시 판돈"은 사행성 규제로 직결 → **판돈을 포인트/명예/배지로 치환**해야 산다.
- **B2B/B2G:** 지자체 걷기 챌린지 + **지역화폐 리워드**는 "건강증진+지역경제"로 관제 예산을 딴다. 광고 없이 기관이 돈을 내는 유일한 모델이나, 영업 사이클이 길다.
- **커머스/O2O:** 걷기를 **오프라인 실제 방문(체크인/CPV)**으로 연결하면, 노출(CPM)보다 비싼 "검증된 방문" 광고를 팔 수 있다. 시니어 지역 상권과 궁합이 좋다.

## 지금 당장 할 것 — 2주 MVP (매몰비용 0)

루프가 세 컨셉 모두에서 공통으로 권한 것: **"수익화는 리텐션이 증명된 다음 문제. 먼저 단 하나의 가설만 코드 거의 없이 검증하라."**

**옵션 A (트래픽 가설 — 명당 웹 공유카드):**
이미 보유한 `data/lucky_stores.json` 명당 DB로 "우리 동네 로또 1등 명당 + 걷기 코스 1개 + 오늘의 운세 한 줄"을 **설치 없이 열리는 서버리스 정적 페이지**로 생성. 카카오 공유 OG 이미지 동적 생성 + "앱에서 QR 자동확인" 딥링크. 측정: 공유클릭·재공유율·페이지→앱 전환. → **"명당 미끼가 정말 카톡 바이럴을 만드는가"**를 검증.

**옵션 B (리텐션 가설 — 걸음 추첨 인앱 실험):**
기존 로또 앱에 '걸음 추첨' 탭 하나만 추가. `expo-pedometer` + `expo-notifications`로 (1) 1만보=응모권 적립, (2) 저녁 8시 추첨 결과 푸시, (3) 상금은 광고 없이 내 주머니에서 소액(주 10만~30만원). 측정: **D7 리텐션과 저녁 8시 재방문율.** → **"변동보상이 고정 100원보다 사람을 다시 부르는가"**를 검증.

**옵션 C (구독 가설 — 하루 한 장 안부 카드):**
부모 설치→걷기→자녀 폰에 하루 1회 안부 카드 자동 도착, 이것 딱 하나. 급감/외출없음은 AI 아닌 룰 기반 임계치(전일 대비 -50%, N일 연속 외출 0)로 하드코딩. 자기 부모+지인 10~20페어에 4주 붙여 시니어 리텐션·카드 개봉률만 측정. → **"자녀가 이 카드에 돈 낼 만큼 안심하는가"**를 검증.

**추천 실행 순서:** 규제 리스크가 없는 **옵션 C(효도)**를 본편으로, **옵션 A(명당 카드)**를 병렬 트래픽 실험으로 동시에 돌린다. 옵션 B(추첨)는 법률·스토어 정책 확인이 끝나기 전에는 착수하지 않는다.

## 방법론 메모
