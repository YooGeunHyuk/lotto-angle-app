주요 기능

• 번호 추천

• 내 번호
구매한 번호를 직접 등록하거나 QR로 스캔해 저장할 수 있습니다.
추첨 결과가 업데이트되면 저장한 번호의 당첨 여부를 확인할 수 있습니다.

• 명당 지도
내 주변 로또 판매점과 당첨 이력이 있는 판매점을 확인할 수 있습니다.
전국 명당 순위, 지역별 명당, 최근 회차 당첨점도 함께 볼 수 있습니다.

• 통계
번호별 출현 빈도, 미출현 번호, 합계, 홀짝, 연속 번호 등 다양한 통계를 확인할 수 있습니다.

## App Privacy

앱이 직접 회원 DB를 운영하거나 로그인을 받지는 않는다. 다만 광고 SDK, 지도, 위치 권한, QR/내 번호 저장 기능이 있으므로 보수적으로 입력한다.

App Store Connect의 `App Privacy`에서 다음 항목을 확인한다.

```text
Contact Info: 수집하지 않음
Health and Fitness: 수집하지 않음
Financial Info: 수집하지 않음
Location: 위치 기능 사용 시 처리 가능
Sensitive Info: 수집하지 않음
Contacts: 수집하지 않음
User Content: QR 인식 결과/구매 번호는 기기 내 저장. 서버 전송 없음
Browsing History: 수집하지 않음
Search History: 수집하지 않음
Identifiers: AdMob 사용으로 광고 식별자/기기 식별자 처리 가능
Usage Data: 광고 노출/반응 및 앱 이용 정보 처리 가능
Diagnostics: 오류/성능 정보 처리 가능
Other Data: 해당 없음
```

### 권장 입력 방향

```text
Location
- Purpose: App Functionality
- Linked to User: No
- Used for Tracking: No

Identifiers
- Purpose: Third-Party Advertising, Analytics 또는 App Functionality
- Linked to User: AdMob 설정에 따라 확인
- Used for Tracking: 개인화 광고/IDFA 사용 시 Yes

Usage Data
- Purpose: Third-Party Advertising, Analytics
- Linked to User: AdMob 설정에 따라 확인
- Used for Tracking: 개인화 광고/IDFA 사용 시 Yes

Diagnostics
- Purpose: App Functionality, Analytics
- Linked to User: 보통 No
- Used for Tracking: No
```

### ATT / Tracking 주의

현재 앱 코드에는 별도 App Tracking Transparency 권한 요청 화면이 없다. 따라서 다음 중 하나로 정리해야 한다.

```text
안전한 방향:
- AdMob을 비개인화 광고 중심으로 설정
- IDFA 기반 개인화 광고/추적을 사용하지 않음
- App Privacy에서 Tracking을 No로 답변
```

또는:

```text
개인화 광고를 사용할 경우:
- ATT 권한 요청 기능 추가 필요
- App Privacy에서 Tracking을 Yes로 답변
- 심사 메모와 개인정보처리방침에도 광고 추적 설명 보강 필요
```

현재 출시 방향은 `비개인화 광고 / Tracking No` 쪽이 더 안전하다.

# 개인정보처리방침 업데이트 여부

현재 개인정보처리방침에는 광고, 위치, 카메라, QR, 내 번호 저장, 지도, 동행복권 조회, GitHub 호스팅이 이미 반영되어 있다.

다만 이번 최종 업데이트 기준으로 아래 내용은 유지/확인한다.

```text
- Google AdMob 사용 명시
- 위치 정보 사용 목적: 내 주변 판매점 및 당첨 이력 판매점 표시
- 카메라 사용 목적: 로또 QR 스캔
- 저장한 구매 번호 및 QR 인식 결과는 기기 내 저장
- 앱은 회원가입 정보를 직접 수집하지 않음
- 앱 내 복권 구매/판매/베팅 기능 없음
```

개인정보처리방침 시행일은 최신 제출일에 맞춰 `2026년 5월 5일`로 업데이트한다.
