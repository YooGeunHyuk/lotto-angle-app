# 맛집 지도 — 데이터 소스 · 합법 아키텍처 · 걷기 연계

> "네이버·카카오·블로그·캐치테이블 맛집을 전국 지도에" 를 **약관·저작권을 지키면서** 구현하는 법.
> 딥리서치(검증 23개 주장)로 확인. ✅=3-표 검증, ◇=일반적으로 알려진 사실(해당 라운드 자동검증 미회수).

---

## 0. 핵심 원칙 (제일 중요)

> **무단 스크래핑으로 "다 긁어서 한 지도에"는 안 된다.** 대신 **공식 API + 공공데이터 = 지도에 직접 표시**,
> **캐치테이블·네이버 플레이스·블로그 = 출처표시 + 링크아웃(딥링크)** 으로 연결한다.

**법적 근거(✅):** 여기어때 vs 야놀자 대법원 판결(2021도1533, 2022.5.12). 여기어때가 야놀자 앱 API에
4개월간 약 1,594만 회 접근해 크롤링 → **최종 무죄**. 그러나 무죄 사유는 "① 정보가 일부에 불과, ② 이미 널리
알려진 정보, ③ 야놀자 이익을 부당히 해치지 않음". **뒤집어 말하면** — 대상이 API를 기술적으로 차단하고
약관에 '무단 크롤링 금지'를 명시했는데 상당한 양질의 데이터를 크롤링하면 **형사처벌 가능성이 매우 높다.**
→ 캐치테이블·네이버를 몰래 긁는 건 회색지대가 아니라 **실질적 리스크**. 우리는 안 한다.

## 1. 데이터 소스별 정리

| 소스 | 무엇을 주나 | 지도 직접표시? | 비고 |
|------|------------|:---:|------|
| **공공데이터 · 전국일반음식점표준데이터** ✅ | 전국 음식점 상호·주소·업종·인허가·영업상태 (무료·전국) | ✅ 가능 | **좌표가 TM(EPSG:5174)** → WGS84 변환 필요. **리뷰·평점 없음** |
| **공공데이터 · MFDS 인허가 업소정보**(15064977) ✅ | 상호·업종·주소·전화·인허가일 (무료 OpenAPI, data.go.kr 키 필요) | ✅ 가능 | 인허가 원장, **리뷰·평점 없음**. 취소·심사중 업소 누락 가능 |
| **서울 열린데이터 OA-16094** ✅ | 서울 음식점 상호·도로명주소·업종·면적 (3일 주기, 오픈라이선스) | ✅ 가능 | 좌표 **EPSG:2097** → 변환 필요 |
| **카카오 로컬 API** ◇ | 키워드/카테고리(FD6 음식점) 검색, 좌표 반환 (전국) | ✅ 가능 | 앱키 발급, 쿼터·상업이용 정책 확인. **리뷰 본문은 API 미제공** |
| **네이버 클라우드 Maps** ✅ | 지도표시·정적지도·지오코딩·역지오코딩·경로(Directions) | 지도 렌더용 | 앞 4개 무료 티어(대표계정), Directions 유료. **플레이스/리뷰 콘텐츠는 안 줌** |
| **네이버 검색 API(지역·블로그)** ◇ | 지역 검색 결과·블로그 글 링크 | 링크아웃 | 표시 항목·재배포 제한 준수. 블로그는 **링크로 연결** |
| **구글 Places API** ◇ | Nearby/Text Search·Place Details(평점·리뷰 일부) | ✅ 가능(유료) | 한국 커버리지 보통, **호출당 과금** — 비용 관리 필수 |
| **캐치테이블 / 다이닝코드 / 망고플레이트** ◇ | 예약·큐레이션 | ❌ (공개 API 없음) | **딥링크/제휴로 연결만.** 무단 크롤링 금지 |

**요점:** *지도에 핀으로 직접 찍는 1차 데이터 = 공공데이터 + 카카오 로컬(+선택적으로 구글 Places).
평점·예약·블로그 후기 = 링크아웃.* 공공데이터는 전국 커버·무료지만 **좌표 변환 + 리뷰 부재**가 한계.

## 2. 지도 렌더링 (지도 SDK)

| 옵션 | 장점 | 주의 |
|------|------|------|
| **카카오맵 JS SDK** ◇ (권장) | 국내 지명·건물 최적, 클러스터러 제공, 무료 쿼터 | 앱키, 도메인 등록, 쿼터 |
| 네이버 지도 JS ✅ | 지오코딩·경로와 통합 | Directions 유료 |
| 구글맵 | 글로벌 | 유료, 국내 디테일 약함 |
| **OSM + MapLibre/Leaflet** ✅ | 비용 통제·자유도 | **자체 타일서버 필요**(기본 tile.osm.org는 개발용만), **ODbL v1.0 — 저작자표시 필수**. 한국은 **osm.kr** 타일 이용 가능 |

→ **국내 서비스는 카카오맵 SDK가 1순위**, 비용·독립성 중시 시 OSM+MapLibre(osm.kr 타일, 저작자표시).

## 3. 다중 소스 통합 아키텍처

```
[공공데이터 원장(전국)] ──┐
[카카오 로컬 검색]      ──┤→ 서버에서 정규화·좌표변환(TM→WGS84)·중복제거 → 우리 DB(장소 마스터)
[구글 Places(선택)]     ──┘        │
                                   ├→ 지도에 핀(클러스터링, 뷰포트 기반 로딩)
[네이버 블로그 후기]  ─→ 링크아웃  │
[캐치테이블 예약]     ─→ 딥링크    │
[사용자 리뷰(자체 축적)] ──────────┘  ← 우리만의 데이터 해자
```

- **중복 제거:** 상호 정규화 + 좌표 근접(예: 30m 이내) 매칭으로 같은 가게 병합.
- **뷰포트 로딩:** 지도에 보이는 영역만 검색/렌더(전국을 한 번에 X).
- **좌표 변환:** 공공데이터 TM(EPSG:5174/2097) → WGS84 필수(proj4 등).
- **리뷰·평점 해자:** 외부 API가 리뷰를 잘 안 주므로, **걷기 유저의 자체 리뷰/인증**을 쌓는 게 장기 차별화.

## 4. 걷기 앱과의 결합 (우리만의 각도)

- **"걸어서 갈 맛집"**: 현재 위치에서 **도보 N분·약 N걸음** 내 맛집만 필터(보행자 경로).
- **맛집까지 걸어가기 챌린지**: 차 대신 걸어서 도착 → 성취 배지·나무 성장(비금전 보상과 연결).
- **미식 산책 코스**: 맛집 2~3곳을 잇는 도보 코스 추천.
- **오운완 + 맛집 인증**: 걷고 나서 맛집 방문을 카드로 (자체 리뷰 축적).
- 이게 캐치테이블·망고플레이트와 다른 점: **"먹으러 걸어가는 경험"** 을 판다.

## 5-A. 실제 카카오맵 켜는 법 (프로토타입 → 배포)

앱에는 **카카오맵 실연동 코드**가 이미 들어 있다(`src/lib/kakaoMap.js`, `components/KakaoMap.jsx`).
키가 없거나(현재 데모) CSP로 스크립트가 막히면 자동으로 **목업 지도로 폴백**한다. 실지도를 켜려면:

1. [Kakao Developers](https://developers.kakao.com)에서 앱 생성 → **JavaScript 키** 발급.
2. 플랫폼 > Web에 **배포 도메인 등록**(로컬은 http://localhost:5173 등).
3. 빌드 환경변수 설정: `VITE_KAKAO_MAP_KEY=<JS키>` (`.env.example` 참고). 또는 호스트 페이지가
   `window.__KAKAO_KEY__`를 주입.
4. 배포하면 KakaoMap이 실제 지도 + **카카오 로컬 keywordSearch('성수동 맛집')** 로 마커·클러스터링 표시.

> ⚠️ 아트팩트(오프라인·CSP)에서는 외부 스크립트가 막혀 **항상 목업**으로 보인다 — 이는 정상 동작이다.

## 6. 하지 않는 것 (준법)

- ❌ 네이버 플레이스·캐치테이블·다이닝코드 **무단 크롤링/재배포** (약관·저작권·형사 리스크).
- ❌ 타 서비스 리뷰 텍스트를 우리 화면에 복제. (링크아웃으로 원출처 방문 유도.)
- ❌ 출처 미표기. (모든 데이터는 출처·저작자 표시 — 공공데이터·OSM·API 각각.)
